## **Introduction/Background:**

Movies and TV shows have wide appeal, and it is important to identify the key factors that contribute to their profitability and success, which benefits both filmmakers and audiences [1]. Previous research has demonstrated that utilizing machine learning techniques to predict box office revenue is effective when incorporating features such as social media engagement, cast and crew, genre, and release date [1]. By doing so, the film industry can gain a better understanding of what factors contribute to a movie's success or failure, leading to improved profitability and higher success rates. 

The aim of this project is to predict the box office performance of movies in terms of revenue generated by analyzing a dataset obtained from Kaggle, which comprises over 3000 data points and more than 20 features including budget, directors, cast, and crew, among others, with the objective of determining the most influential variables. Additionally, we aim to extend this approach to develop a recommendation system that suggests movies based on an individual's mood in the future [2] [4]. 

## **Problem Definition:**

This project aims to use predictive modeling techniques to benchmark the accuracy of box office revenue predictions for movies [2] [3]. By analyzing various features of movies, such as genre, budget, cast, release date, and popularity, we aim to develop and compare different predictive models. The objective is to identify the most reliable and effective models that can assist the film industry in making informed decisions about budget, popularity and cast.

We will be demonstrating how to use supervised and unsupervised models on the dataset and report our findings on what contributes to a model's success and which types of models are most effective in terms of accurately predicting box office revenue. 
## **Dataset:**

The objective of our project is to compare and evaluate different movie recommendation algorithms using a dataset available on [Kaggle](https://www.kaggle.com/competitions/tmdb-box-office-prediction). The training set of the database contains information about 3000 movies, including budget, cast, crew, release date, and popularity, among others. The test set on the other hand, contains information about 4398 movies, but does not include the box office revenue. Both files are in CSV format, which makes it easy to import and analyze the data.

### **Data Collection and Description:**

As mentioned in the previous section, our dataset was sourced from a Kaggle competition which took place between February and May 2019. As part of this competition, the participants were task with predicting the overall worldwide box office revenue for the films selected from the The Movie Database. The dataset contained information on cast, crew, plot keywords, budget, posters, release dates, spoken languages, production companies and countries, and several other key features of the movies. The evaluation metric of choice for the competition was Root-Mean-Squared-Logarithmic-Error (RMSLE) between the predicted value and the actual revenue. 

The original dataset contained in excess of 7000 movies with 4398 of theses movies belonging to the test set and hence not containing revenue values. This was an issue as we wanted to combine the datasets and randomly sample to generate our own training and testing sets for model building.

### **1. Multiple Linear Regression:**

#### **1.1. Forward Selection and Backward Stepwise Regression:**


We first performed linear regression on the dataset with the aim of getting a baseline model to use as a benchmark for the other models we would be developing. We used the statsmodels library to perform linear regression on the dataset. We used the r-squared value to measure the model's accuracy. 

In addition to performing simple multiple linear regression (MLR), we used our script to perform forward selection and backward elimination to select the best features for our model. At each step, we used Akaike Information Criterion (AIC) values to get an understanding of model performance, while also considering using a metric such as adjusted r-squared. 

```python
import pandas as pd
import numpy as np
from sklearn.linear_model import LinearRegression
from sklearn.feature_selection import RFE
from sklearn.metrics import r2_score
import statsmodels.api as sm

def forward_selection_aic(X, y, max_features=None):
    if max_features is None:
        max_features = X.shape[1]
    selected_features = []
    best_aic = float('inf')
    for i in range(max_features):
        remaining_features = [f for f in X.columns if f not in selected_features]
        aic_values = []
        for f in remaining_features:
            model = sm.OLS(y, sm.add_constant(X[selected_features + [f]])).fit()
            aic = model.aic
            aic_values.append((aic, f))
        aic_values.sort()
        if aic_values[0][0] < best_aic:
            selected_features.append(aic_values[0][1])
            best_aic = aic_values[0][0]
        else:
            break
    return X[selected_features]

def backward_elimination_aic(X, y, sig_level=0.05):
    X = sm.add_constant(X)
    model = sm.OLS(y, X).fit()
    n_features = X.shape[1] - 1
    best_aic = model.aic
    while model.pvalues.max() > sig_level and n_features > 0:
        feature_to_remove = model.pvalues.idxmax()
        X = X.drop(feature_to_remove, axis=1)
        model = sm.OLS(y, X).fit()
        n_features -= 1
        if model.aic < best_aic:
            best_aic = model.aic
        else:
            X = X.drop(X.columns[-1], axis=1)
            break
    return X.iloc[:, 1:]

def evaluate_model(model, X, y):
    y_pred = model.predict(sm.add_constant(X))
    residual = y - y_pred
    rss = np.sum(residual ** 2)
    tss = np.sum((y - np.mean(y)) ** 2)
    r2 = 1 - rss / tss
    n = len(y)
    k = X.shape[1]
    aic = n * np.log(rss / n) + 2 * k
    return r2, aic
```

### **4. KMeans:**

K-means is a popular unsupervised machine learning algorithm used for clustering data points. The algorithm groups data points into a specified number (k) of clusters based on their similarities in features. K-means works by iteratively assigning data points to the nearest cluster center (centroid) and then recalculating the centroid based on the new assignments. This process continues until the centroids no longer change significantly or a maximum number of iterations is reached.

Here, K-means has been applied to group similar movies into clusters based on their features such as genre, popularity, rating, and cast. By doing so, it can help in movie recommendation systems and marketing by identifying groups of movies that are popular with similar audiences. For example, a movie streaming platform can use K-means to cluster movies that are popular among a certain age group or demographic and recommend similar movies to users based on their viewing history.

### **4.1. Results and Discussion for KMeans Clustering:**

#### **4.1.1. DB-Index:**

The Davis-Bouldin Index (DBI) is a clustering evaluation metric that measures the separation between clusters and the similarity within clusters. It is calculated as the ratio of the within-cluster distance to the between-cluster distance, averaged over all clusters. A lower value of the DBI indicates a better clustering result.

The dataset shows that the DBI value is at its lowest when the number of clusters is either 2 or 3, which suggests that these numbers of clusters are the optimal choices for this dataset.

```python
from sklearn.cluster import KMeans
from sklearn.metrics import davies_bouldin_score

db_scores = []

for i in range(2, 21):s
    kmeans = KMeans(n_clusters=i, random_state=0)
    labels = kmeans.fit_predict(subset_df)
    db_score = davies_bouldin_score(subset_df, labels)
    db_scores.append(db_score)

plt.plot([int(x) for x in range(2, 21)], db_scores)
plt.xlabel('Number of clusters')
plt.ylabel('Davies-Bouldin score')
```

#### **4.1.2. Silhouette Coefficient:**

The silhouette coefficient is a measure of how similar an object is to its own cluster compared to other clusters. It is a value between -1 and 1, with values closer to 1 indicating that the object is well-matched to its own cluster and poorly-matched to neighboring clusters, and values closer to 0 indicating that the object is equally well-matched to its own cluster and neighboring clusters. Negative values indicate that the object is poorly-matched to its own cluster and well-matched to neighboring clusters.

By calculating the silhouette coefficient for various numbers of clusters in the movie dataset, it was possible to identify the most suitable number of clusters for grouping the movies based on their attributes. The analysis revealed that the optimal number of clusters for this dataset is either 2 or 3.

```python
from sklearn.metrics import silhouette_samples, silhouette_score

cluster_range = range(2, 21)
silhouette_scores = []
for n_clusters in cluster_range:
    kmeans = KMeans(n_clusters=n_clusters, random_state=0)
    labels = kmeans.fit_predict(subset_df)
    silhouette_avg = silhouette_score(subset_df, labels)
    silhouette_scores.append(silhouette_avg)

plt.plot(cluster_range, silhouette_scores)

plt.ylabel("Average silhouette score")
plt.xticks(range(2, 21))
```

#### **4.1.3. Elbow Method:**

The Elbow method is a very popular technique and the idea is to run k-means clustering for a range of clusters k (let’s say from 1 to 10) and for each value, we are calculating the sum of squared distances from each point to its assigned center(distortions).

When the distortions are plotted and the plot looks like an arm then the “elbow” (the point of inflection on the curve) is the best value of k.
